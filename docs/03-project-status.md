## 已确认、待系统化实现的界面设计

### 五种玻璃主题

后续设置系统需要保留 5 种可切换的玻璃主题。主题只改变外层面板材质，
不得改变账户状态色、Hero 语义色、进度条颜色或信息层级。

| 主题标识建议 | 显示名称 | 设计方向 | 默认 |
| --- | --- | --- | --- |
| `crystalGlass` | 冰晶玻璃 | 冷蓝灰、清透、折射明显 | 否 |
| `softPearlGlass` | 柔灰珍珠 | 暖灰白、柔和、低刺激 | 否 |
| `seaGlass` | 海盐玻璃 | 浅青绿、清爽、低饱和 | 否 |
| `moonGlass` | 月光玻璃 | 蓝紫灰、冷静、科技感 | 否 |
| `brightPearlGlass` | 明亮珍珠 | 暖白、轻折射、高明度 | 是 |

实现注意事项：

- 默认使用当前的明亮珍珠玻璃；
- 主题选择需要持久化，并跟随同一账户跨启动生效；
- 浅色、深色模式分别维护语义 Token，不能简单反色；
- 使用原生材质时需要支持“降低透明度”辅助功能；
- 主题切换只做短暂交叉渐变，不使用位移或缩放；
- 当前阶段不增加主题入口，等设置、持久化和 Design Token
  系统一起实现。

### 两种面板模式

以下名称仅作为产品和代码沟通用语，不直接显示在应用界面：

1. **排列模式**：顶部显示选中账户的 Hero，下面最多同时显示 3 个账户；
   账户区域可滚动，账户行可以拖动改变顺序。
2. **卡片模式**：双击 Hero 后，Hero 填满整个面板；通过上下或左右滑动
   切换账户，再次双击返回排列模式。

模式动效约束：

- 排列模式进入卡片模式：Hero 的颜色表面快速扩展至整个窗口，
  Header、账户列表和 Footer 同时逐渐淡出；
- 卡片模式返回排列模式：颜色表面从窗口边缘收缩回 Hero 原位，
  其他内容只通过透明度恢复，不使用移动入场；
- 账户切换使用克制的交叉淡化与数值过渡；
- 所有模式动效必须响应系统“减少动态效果”设置；
- 拖拽排序和账户选择状态需要为后续持久化预留稳定 ID。
