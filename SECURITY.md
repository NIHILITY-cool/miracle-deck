# Security Policy

## Reporting a vulnerability

Please do not open a public issue for credential exposure, authentication
bypass, unsafe URL handling, or another vulnerability that could put users at
risk. Until a private GitHub security contact is configured, contact the
maintainers privately and provide the smallest reproducible example possible.

Do not include real API keys, cookies, OAuth tokens, account exports, prompts,
or model responses in a report.

## Supported versions

The project is pre-release. Security fixes will be applied to the latest
development branch until the first public release defines a stable support
window.
